# 提示词质量清单

维护者和贡献者可以用这份清单快速判断一条提示词是否值得收录。

## 必须满足

- 使用中文为主，允许少量必要英文品牌、参数或技术词。
- 明确用途和画幅比例。
- 有可识别主体，而不是只有抽象风格。
- 至少描述构图、场景、光线或材质中的三项。
- 如果要求生成中文文字，文字内容必须短且完整。
- 包含限制条件，例如错误文字、产品形变、背景杂乱、肢体变形。

## 建议满足

- 提示词可替换变量后复用。
- 描述中包含信息区或留白位置。
- 商业用途说明产品占比、角度、材质和背景复杂度。
- 人像用途说明年龄段、服装、姿态、表情和镜头语言。
- 场景用途说明空间功能、视角高度、动线和人群密度。

## 结果图检查

- 图片内容与原提示词的主体、比例、构图和限制条件一致。
- 文件使用 `.webp`，长边不超过 1600px，路径位于 `assets/cases/`。
- `outputs` 字段记录模型、路径、许可、署名和备注。
- 生成图不包含第三方真实品牌、受版权保护角色、真实人物肖像或水印。
- 转载图必须来自明确兼容许可的 GitHub 来源，并保留来源链接、许可和署名。
- 含中文文字的图片需要人工确认文字可读；明显错误文字不作为样板图。

## 不建议收录

- 只堆叠风格词，没有清楚画面。
- 强依赖某个受版权保护的角色、画师或品牌视觉。
- 要求生成可误导的真实人物、证件、票据或医疗诊断图。
- 包含歧视、骚扰、违法或明显伤害性内容。
- 中文文字过多，导致生成结果很难稳定。
